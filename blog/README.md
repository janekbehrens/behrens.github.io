# Blog — How to publish a new article

The blog index (`index.html`) is **fully dynamic**: it fetches `posts.json` at
runtime and renders the article list from that. There are no hardcoded links.
`posts.json` is generated by `build-index.js`, which scans every `*.html` file
in this directory and extracts metadata automatically.

This means you can never publish a broken link: `posts.json` only contains
articles whose files actually exist, and both are always pushed together.

---

## Step-by-step: publish a new article

### 1 — Create the article HTML file

Copy an existing article as a starting point, e.g.:

```bash
cp wsjf-jira.html my-new-article.html
```

The filename becomes the URL: `https://janekbehrens.de/blog/my-new-article.html`.
Use lowercase, hyphen-separated words. No spaces, no underscores.

---

### 2 — Fill in the required metadata

The build script reads three things from the HTML `<head>`:

#### `<title>` — becomes the article title in the index
```html
<title>Your Article Title Here | Janek Behrens</title>
```
The " | Janek Behrens" suffix is stripped automatically.

#### `<meta name="description">` — becomes the excerpt in the index
```html
<meta name="description" content="2–3 sentence summary shown in the article list. Make it specific and useful.">
```

#### JSON-LD `BlogPosting` schema — provides date and tags
```json
{
  "@type": "BlogPosting",
  "datePublished": "2026-04-10T00:00:00+02:00",
  "keywords": ["jira", "backlog prioritization", "WSJF", "Atlassian"]
}
```

- `datePublished`: ISO 8601 date. Controls sort order in the index.
- `keywords`: array of strings. Controls which filter tabs the article appears under (see tag rules below).

Also update `<link rel="canonical">` and all other URLs inside the JSON-LD
blocks to point to the correct article URL.

---

### 3 — Understand automatic tag detection

`build-index.js` maps keywords to filter categories using these rules
(checked in order; an article can have multiple tags):

| Filter tab | Triggered when keywords/title/filename contain… |
|---|---|
| `priorityscoring` | wsjf, rice, ice, priorit, backlog health, grooming, scoring, prioritization framework |
| `confluence` | confluence |
| `enterprise` | enterprise, byok, compliance, security, gdpr |
| `jira` | jira, sprint, epic, scrum, backlog, agile, safe, pi planning |
| `forge` | forge, atlassian marketplace, ui kit |

If no rule matches, the article gets the `forge` tag as fallback.

The display label shown on the card (e.g. "Jira · Priority Scoring") is built
automatically from the detected tags.

---

### 4 — Regenerate `posts.json`

Run this from the repository root or the `blog/` directory:

```bash
node blog/build-index.js
```

The script prints every detected article with its date and tags. Verify that
your new article appears with the correct tags before pushing.

---

### 5 — Commit and push everything together

Always push the article file and the updated `posts.json` in the same commit:

```bash
git add blog/my-new-article.html blog/posts.json
git commit -m "Blog: add article my-new-article"
git push
```

Do **not** push `index.html` alone with new article entries — `index.html`
contains no article data anymore. Do **not** push an article file without
regenerating `posts.json` first, or the article will be invisible on the site.

---

## File roles at a glance

| File | Role | Edit manually? |
|---|---|---|
| `index.html` | Blog listing page — fetches and renders `posts.json` | Only for layout/style changes |
| `posts.json` | Article index — auto-generated, do not edit by hand | Never (run build-index.js) |
| `build-index.js` | Generator script — reads all `*.html`, writes `posts.json` | Only to adjust tag detection rules |
| `my-article.html` | The article itself | Yes — this is the content |

---

## How the index page renders articles

`index.html` runs `fetch('/blog/posts.json')` on load, then builds `<li>`
elements for each post. The existing search box, filter buttons (All / Jira /
Priority Scoring / Confluence / Enterprise), sort toggle, and pagination all
operate on those dynamically created elements — no page reload needed.

If `posts.json` cannot be fetched (e.g. network error), the page shows the
empty-state message instead of crashing.
